* Feature **#5567** : Stop the content index rebuilding on first use after a node reboot. Add config props `stroom.contentInde.contentIndexDir` (defaults to `content_index`) and `stroom.contentInde.storageType` (one of `TEMP|LOCAL|SHARED`, defaults to `LOCAL`). Thus the content index can now be stored locally on each node for better performance or on shared storage. Stroom now eagerly builds the content index on boot if the storage type is `SHARED`.
